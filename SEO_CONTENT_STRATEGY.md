### 4.3 Header Structure (H1/H2/H3)

**Universal rule:**
- H1: Exact match or near-match to target keyword (one per page)
- H2: Semantic variations and related terms
- H3: Supporting questions or sub-points
- Never use H4-H6 as primary structure

**Example for bail meublé page:**
```
H1: Modèle de bail de location meublé 2026
H2: Bail meublé : durée, clauses obligatoires et modèle gratuit
H3: Quelle durée pour un bail meublé ?
H3: Que doit contenir un bail meublé ?
H3: Dépôt de garantie en bail meublé : règles 2026
H2: Télécharger notre modèle de bail meublé
H3: Comment personnaliser le modèle ?
H3: Les pièges à éviter lors de la rédaction
H2: Questions fréquentes sur le bail meublé
```

### 4.4 Internal Linking Strategy

**Architecture:**
```
HOMEPAGE
├── /gestion-locative/ (Pillar)
│   ├── /outils/modele-bail-location/ (Tool → links to pillar)
│   ├── /quittances/ (Feature page → links to pillar)
│   └── /blog/gestion-locative-definitive-guide (TOFU → pillar)
│       └── (Each blog article links to: pillar + 2 related tools)
│           ├── /outils/calculateur-depot-garantie/
│           └── /outils/calculateur-irl-2026/
├── /pricing/ → links to all tools
├── /glossaire-immobilier/ (glossary hub)
│   └── Each term links to related article + tool
└── /demo/ (conversion page)
```

**Internal link density:**
- Articles: 3-5 internal links per 500 words
- Tool pages: 2-3 contextual links to articles
- Homepage: 8-12 links to category pages
- Glossary: each term links to 1 article + 1 tool

**Anchor text rules:**
- Use exact match keyword as anchor ≤40% of the time
- Use partial match or natural language the rest
- Never "click here" or "read more"
