## Overview

This document defines the operational system for planning, producing, publishing, and maintaining all SEO content at RentReady. It operationalizes the content-briefs.md and content-clusters.md already in /docs.

**Scope:** Blog articles, template pages, calculator pages, guide pages, comparison pages, city pages, pillar pages.
**Goal:** Consistent publication cadence, quality control, SEO compliance, and efficient handoff between PM → SEO Content Lead → Writers → Engineering.

## 4. Writer Guidelines

### Standards for All Writers

**Language:** French only. No machine translation. Native-level French required.
**Tone:** Professional but accessible. Like a knowledgeable friend who happens to be an expert. Not academic, not salesy.
**Style:** Clear sentences, short paragraphs (2-4 sentences max), bullet points for lists, tables for comparisons.

### Content Rules

1. **No thin content.** Minimum 800 words for articles, 1,200 for pillars, 1,500 for calculators. If a topic can't be covered well in that length, flag to PM.

2. **Cite authoritative sources.** Link to:
   - Service-Public.fr (official French government)
   - Legifrance.gouv.fr (French law)
   - INSEE.fr (IRL indices, statistics)
   - ANIL.org / France Rénov' (housing advice)
   - Notaires de France

3. **Always include FAQ section.** Minimum 4 questions. Use actual PAA queries from Google. Do not invent questions.

4. **Internal links are mandatory.** Every article must link to at least 2 existing pages. Use contextual anchor text, not "click here."

5. **No duplicate content.** Each page must have a unique value proposition. If two pages target overlapping keywords, flag to PM for clarification.

6. **Legal sensitivity.** Any content that could be construed as legal advice must include a disclaimer: *"Cet article est informatif et ne constitue pas un conseil juridique. Consultez un professionnel pour votre situation spécifique."*

7. **Images.** Writers do NOT source images. Flag needed images to PM with description. Engineering handles image sourcing and OG card generation.

8. **Calculator/Tool pages.** Writer provides all text content, FAQ questions, and explanatory sections. Engineering implements the interactive logic.

### Writer Onboarding Checklist
Before their first brief, each writer must receive:
- [ ] Link to content-clusters.md
- [ ] Link to content-briefs.md (first 20 briefs as examples)
- [ ] Link to writer guidelines (this section)
- [ ] One trial brief to complete before paid work begins
- [ ] Contact for questions: PM

### Briefing a Writer
PM sends the writer:
1. Completed content brief (above template)
2. Relevant cluster document section
3. Deadline
4. Any special instructions

### Revisions
- Writers get ONE round of revisions included in base rate.
- Major structural changes after first draft = additional round = notify PM.
- PM approves all revisions before writer proceeds.

## 6. Approval Workflow

### Standard Flow (Non-Urgent)

```
[1] PM creates brief
        ↓
[2] SEO Content Lead reviews brief (keyword targeting, cluster logic)
        ↓
[3] Writer receives brief → produces draft
        ↓
[4] PM reviews draft (structure, accuracy, tone)
        ↓
[5] If legal/compliance content → Legal review (1-2 days)
        ↓
[6] Writer implements revisions (1 day)
        ↓
[7] SEO Content Lead reviews for SEO compliance
        ↓
[8] Engineering builds page (if template/calculator/tool)
        ↓
[9] PM final sign-off
        ↓
[10] Publication scheduled (Contentful CMS or direct CMS publish)
        ↓
[11] PM notifies relevant team (Slack/email)
```

### Timeline Standards
| Stage | SLA |
|-------|-----|
| Brief creation | 1 business day |
| Writer draft | 4 business days |
| PM review | 1 business day |
| Legal review | 2 business days (if needed) |
| Writer revisions | 1 business day |
| SEO review | 1 business day |
| Engineering build | 3–5 business days |
| Final PM sign-off | 1 business day |
| **Total (no legal)** | **~10 business days** |
| **Total (with legal)** | **~12 business days** |

### Expedited Flow (High-Priority)
For time-sensitive content (e.g., sudden regulatory change):
1. PM marks issue as "expedited" and notifies all stakeholders.
2. Engineering prioritized for builds.
3. Writer delivers draft in 2 business days.
4. All reviews happen same day.
5. Target: live within 5 business days of brief creation.

## 7. CMS Integration

### Current Setup
Content is managed in **Contentful** CMS (or equivalent Next.js-based CMS in the /home/ubuntu/rent-ready project).

### CMS Content Model
Each content entry has:
| Field | Type | Required | Notes |
|-------|------|---------|-------|
| title | Text | Yes | H1, used for SEO title if not set |
| slug | Text | Yes | URL path, must be unique |
| content_type | Enum | Yes | article, template, calculator, pillar, comparison, city |
| cluster | Enum | Yes | 1-5 from content-clusters.md |
| target_keyword | Text | Yes | Primary keyword |
| secondary_keywords | List<Text> | No | Up to 3 |
| meta_title | Text | Yes | <60 chars |
| meta_description | Text | Yes | 155-160 chars |
| body | RichText | Yes | Main content |
| faq | JSON | Yes | Array of {question, answer} |
| author | Reference | Yes | Author profile |
| featured_image | Media | No | For article templates |
| og_image | Media | Yes | 1200×630, text overlay |
| schema_markup | JSON | Yes | Auto-generated from fields |
| internal_links | JSON | Yes | {url, anchor, position} array |
| publish_date | Date | Yes | Scheduled publish date |
| review_date | Date | Yes | 12 months from publish |
| status | Enum | Yes | draft, in_review, approved, published |

### Publishing Checklist (CMS)
- [ ] Slug matches target URL from brief
- [ ] Meta title and description set and under limits
- [ ] Primary keyword in H1, meta title, first paragraph
- [ ] FAQ section added with 4+ PAA questions
- [ ] Internal links placed (minimum 3)
- [ ] External links to authoritative sources
- [ ] OG image uploaded and approved
- [ ] Schema markup validated (Google Rich Results Test)
- [ ] BreadcrumbList configured
- [ ] Canonical URL set
- [ ] Robots meta: index, follow (unless intentionally noindex)
- [ ] Content reviewed for duplicate content risk
- [ ] Publish date scheduled (avoid Mondays — target Wed/Thu)
- [ ] Review date set in CMS calendar (12 months out)

## 8. Content Refresh & Maintenance

### Quarterly Content Audit
Every 90 days, PM and SEO Content Lead review:
1. **Ranking changes** — pages that dropped >5 positions for target keywords
2. **Traffic changes** — pages with >20% traffic decline
3. **Freshness signals** — articles older than 18 months without updates
4. **Schema errors** — any pages failing structured data tests
5. **Broken links** — external links that changed or 404'd
6. **Regulatory changes** — any legal content that may be outdated

### Annual Full Review
- All pillar pages: full rewrite or substantial update
- All articles: update statistics, add new sections if needed, refresh examples
- Calculators: verify calculation logic is still correct (IRL values, legal thresholds change)
- Templates: verify still legally compliant

### Content Retirement
If a page has no traffic after 6 months and no strategic value:
1. PM proposes redirect to nearest relevant page
2. SEO Content Lead approves redirect
3. Engineering implements 301 redirect
4. Old page removed from sitemap
5. Document in content audit log
