## Internal Linking Strategy

### Hub-and-Spoke Model

Each content cluster has a **pillar page** (hub) linked to by all supporting articles (spokes).

**Pillar pages:**
1. `/blog/gestion-locative-guide-complet` → spokes: all 5 cluster articles
2. `/glossaire` → spokes: all glossary terms
3. `/outils/calculateur-irl` → spokes: revision, rentabilité, charges calculators
4. `/pour-les-proprietaires` → spokes: case study, comparison, tools pages
5. `/blog/bail-meuble-vs-bail-vide` → spokes: bail mobility, bail saisonnier, colocation

**Link placement rules:**
- Every article: 2-3 links to pillar pages + 2-3 links to related articles
- Every tool page: link to 2 related blog articles
- Every ICP page: link to 2-3 relevant blog articles + tool pages
- Every PDF template: links back to the explaining blog article

## SEO Quality Gates (per piece before publish)

Before any content goes live, verify:
- [ ] Target keyword in H1, first 100 words, and at least one subheading
- [ ] Meta title ≤ 60 chars, includes primary keyword
- [ ] Meta description ≤ 155 chars, includes CTA + keyword
- [ ] Word count meets minimum (see table above)
- [ ] At least 2 internal links to existing content
- [ ] At least 1 link to an external authoritative source (government, official)
- [ ] Schema markup: Article, FAQPage, or HowTo as appropriate
- [ ] Social share image generated (1200x630)
- [ ] Table of contents for articles >1500 words
- [ ] Reading time displayed (target: 5-8 min for articles)
