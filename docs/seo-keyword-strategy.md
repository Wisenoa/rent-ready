### 4.2 Structure des En-têtes (H1-H6)

```
H1: Mot-clé principal (1 seul par page)
H2: Sections principales (3-6 par page)
H3: Sous-sections (2-4 par H2)
H4: Détails spécifiques (si nécessaire)
H5-H6: Rarement utilisés
```

**Règles:**
- H1 contient toujours le mot-clé principal
- H2 contiennent des variations du mot-clé
- H3 répondent aux questions connexes (People Also Ask)
- Pas de saut de niveau (H1 → H3 interdit)

### 4.3 Stratégie de Maillage Interne

**Règles de linking:**

1. **Pillar pages** → 10-15 liens vers articles cluster
2. **Articles cluster** → 1 lien vers pillar + 3-5 liens vers articles connexes
3. **Template pages** → 2-3 liens vers articles explicatifs + 1 CTA vers produit
4. **Calculateurs** → 2-3 liens vers guides connexes + CTA vers essai gratuit
5. **Glossaire** → Chaque terme lié à la page pertinente

**Hub pages:**
- `/gestion-locative` → Hub principal (tous les guides liés)
- `/blog` → Hub de contenu éducatif
- `/outils` → Hub d'outils et templates
- `/glossaire-immobilier` → Hub de définitions

### 4.4 Directives pour les Images

**Alt text:**
- Descriptif et naturel
- Inclure le mot-clé si pertinent
- Maximum 125 caractères
- Exemple: `alt="Exemple de quittance de loyer générée par RentReady"`

**Optimisation technique:**
- Format WebP par défaut
- Taille max: 100 KB pour les photos, 50 KB pour les icônes
- Dimensions explicites (width/height) pour éviter le CLS
- Lazy loading pour les images hors viewport
