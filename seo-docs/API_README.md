## SEO Analytics Events API

### POST /api/seo/events
Server-side event logging for SEO analytics.

**Body**
```json
{
  "eventType": "template_download | blog_cta_click | calculator_use | page_view",
  "resourceSlug": "bail-vide",
  "pageUrl": "/templates/bail-vide",
  "sessionId": "sess_abc123",
  "referrer": "https://google.fr",
  "metadata": { "extra": "info" }
}
```

**Required fields:** `eventType`, `pageUrl`

**Response:** `201 Created` — `{ ok: true, id: "cuid..." }`

**Privacy:** IP addresses are hashed with SHA-256 before storage.

### GET /api/seo/events
Query SEO analytics events (admin-only, requires `x-seo-api-key` header).

**Headers:** `x-seo-api-key` must match `SEO_ANALYTICS_API_KEY` env var

**Query params:**
- `eventType` — filter by event type
- `resourceSlug` — filter by resource
- `from` — ISO date string (start)
- `to` — ISO date string (end)
- `limit` — default 100, max 1000

**Response**
```json
{
  "events": [{ "id": "...", "eventType": "template_download", "resourceSlug": "bail-vide", "pageUrl": "/templates/bail-vide", "createdAt": "..." }],
  "total": 42
}
```

- All glossary and property-type endpoints are **public** (no auth required) — they serve marketing/SEO pages
- Structured data endpoints return valid JSON-LD compatible with Google's Rich Results Test
- The `updatedAt` field is surfaced on all content endpoints for sitemap `lastmod` support
- Glossary data lives in `/src/data/glossary.json` — add new terms there
- Property type data lives in `/src/data/property-types.ts` — edit there
