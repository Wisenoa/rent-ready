## SECTION 5: CMS CONTENT MODEL REQUIREMENTS

To scale content production beyond 100 articles, the CMS content model must support:

### Content Types

#### 1. Article (Blog)
```
Article {
  id: UUID
  slug: string (unique, URL-safe)
  locale: 'fr' | 'en'
  status: 'draft' | 'review' | 'published' | 'archived'
  
  // SEO
  title: string (max 70 chars)
  metaDescription: string (max 155 chars)
  focusKeyword: string
  secondaryKeywords: string[]
  
  // Content
  excerpt: string (max 300 chars)
  content: Markdown
  readTimeMinutes: number
  featuredImage: Image
  
  // Taxonomy
  category: Category (Gestion/Bail/Loyers/Entretien/Fiscalite/Outils)
  tags: Tag[]
  cluster: ContentCluster (1-5)
  
  // Authorship
  author: Author (name, role, avatar)
  publishedAt: DateTime
  updatedAt: DateTime
  
  // Schema
  schemaType: 'Article' | 'BlogPosting'
  faqItems: FAQItem[]
  
  // Links
  internalLinks: { targetSlug: string, anchor: string }[]
  externalLinks: { url: string, label: string }[]
  
  // SEO flags
  noIndex: boolean
  canonicalUrl: string
}
```

#### 2. Guide (long-form reference page)
```
Guide {
  ...all Article fields...
  
  // Additional for guides
  type: 'pillar' | 'topic' | 'procedure'
  isPillar: boolean
  stepCount: number
  estimatedTime: string ("15 minutes")
  
  // HowTo specific
  howToSteps: { name: string, text: string, image?: Image }[]
  howToTotalTime: string
  
  // Breadcrumb override
  breadcrumbLabel: string
}
```

#### 3. Tool/Outil (calculator pages)
```
Tool {
  ...all Article fields...
  
  calculatorType: 'irl' | 'charges' | 'depot' | 'preavis' | 'rendement' | 'revision' | 'surface' | 'plus_value'
  inputFields: { name: string, label: string, type: 'number' | 'text' | 'select', required: boolean, options?: string[] }[]
  outputFields: { name: string, label: string, format: 'currency' | 'percent' | 'date' | 'text' }[]
  formula: string (for documentation)
  
  toolCTA: { text: string, url: string }
}
```

#### 4. Template (document download pages)
```
Template {
  ...all Article fields...
  
  documentType: 'bail' | 'quittance' | 'etat-des-lieux' | 'relance' | 'conge' | 'autre'
  downloadUrl: string
  previewUrl: string
  fileFormat: 'PDF' | 'DOCX' | 'ODS'
  isFree: boolean
  rentReadyBranding: boolean
  
  relatedTools: Tool[]
  legalReferences: string[]
}
```

### Required CMS Capabilities

| Capability | Priority | Description |
|-----------|----------|-------------|
| Markdown editor with preview | Critical | Writers need rich text + raw markdown |
| FAQ JSON-LD editor | Critical | Structured FAQ items with auto-schema |
| Internal link picker | High | Browse-and-select for internal links |
| Slug auto-generation | High | From title, with override |
| Scheduling | High | Publish date/time |
| Canonical URL field | High | For avoiding duplicate content |
| Redirect management | Medium | For slug changes |
| Revision history | Medium | Track all changes |
| A/B title/meta testing | Low | For conversion optimization |

```
POST /api/content/articles     — create article

PATCH /api/content/articles/:id — update article
POST /api/content/articles/:id/publish — publish article
```

### Quality Gates

Every article must pass:
1. Word count check (>= target)
2. Keyword placement check (H1, intro, 2+ H2s)
3. Internal link check (2+ existing pages)
4. External link check (1+ official source)
5. Schema check (FAQPage + Article/Guide + BreadcrumbList)
6. Alt-text check (all images)
7. Readability check (70+ Hemingway score)
8. Meta check (title 60 chars, description 155 chars)
